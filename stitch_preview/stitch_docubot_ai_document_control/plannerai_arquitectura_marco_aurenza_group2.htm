PlannerAI

Arquitectura Marco para MVP SaaS

Herramienta IA para transformar documentos de alcance contractual en cronogramas Gantt, escenarios y exportaciones Primavera P6 / MS Project

 

Cliente objetivo: Aurenza Group
Industria: staffing especializado, mineria, construccion, ingenieria y proyectos EPC/EPCM en Chile
Stack propuesto: Python FastAPI, React, LangChain, PostgreSQL, Azure, GPT-4o

 

 

1. Resumen ejecutivo

PlannerAI es un MVP SaaS orientado a empresas de staffing profesional y soporte tecnico para mineria y construccion. Su proposito es convertir documentos de alcance contractual en una estructura planificable: hitos, entregables, restricciones, actividades, dependencias, escenarios y ruta critica.

El producto debe operar como copiloto experto, no como caja negra. La IA propone, el usuario valida, y el sistema conserva trazabilidad documental para auditar cada actividad generada contra su fuente contractual.

Elemento

Definicion para el MVP

Objetivo

Reducir el tiempo de lectura y estructuracion de alcances contractuales, generando cronogramas editables y exportables.

Usuario principal

Planner, PMO, administrador de contrato, jefe de servicio, lider de propuesta o consultor Aurenza.

Entrada

Documento de alcance en PDF o DOCX; idealmente bases tecnicas, SOW, contrato, anexo tecnico o especificacion.

Salida

Gantt interno, tres escenarios, ruta critica, matriz de restricciones, supuestos, riesgos y XML compatible con P6/MS Project.

Principio clave

Human-in-the-loop: toda inferencia de IA debe quedar marcada y disponible para revision.

4. Flujo funcional de punta a punta

Figura 2. Flujo funcional del MVP.

1.     El usuario crea un proyecto, define mandante, nombre del contrato y fecha tentativa de inicio.

2.     El usuario carga el documento de alcance contractual en PDF o DOCX.

3.     El backend extrae texto. Si el documento es escaneado, se activa OCR.

4.     LangChain invoca GPT-4o con un prompt de extraccion estructurada y esquema JSON.

5.     El sistema valida el JSON, marca inferencias y presenta resultados para revision humana.

6.     El motor de planificacion genera actividades, dependencias, fechas y ruta critica.

7.     El sistema construye escenarios optimista, base y conservador.

8.     El usuario exporta a XML compatible con Primavera P6 o MS Project.

12. Diez decisiones tecnicas clave antes de empezar

Nivel de trazabilidad documental

Definir si cada actividad tendra pagina, parrafo, clausula, cita textual y score de confianza. Recomendacion: si, desde el MVP.

MVP automatico o human-in-the-loop

Definir si la IA genera cronogramas finales o borradores revisables. Recomendacion: borrador IA + validacion humana.

Formato de salida principal

Priorizar Gantt interno + MS Project XML + Primavera P6 XML. Dejar MPP nativo para etapa posterior.

Motor de planificacion

Decidir entre motor propio o integracion. Recomendacion: motor propio liviano en Python con NetworkX/CPM.

Manejo de incertidumbre

Toda duracion o dependencia inferida debe marcarse como inferred y requerir revision.

Estrategia IA

Evitar un unico prompt gigante para documentos extensos. Recomendacion: chunking, extraccion parcial, consolidacion y validacion JSON.

OCR y documentos escaneados

Incorporar Azure Document Intelligence como fallback para PDFs escaneados.

Multi-tenancy

Disenar multi-tenant desde el inicio, aunque el primer despliegue sea para Aurenza solamente.

Compatibilidad P6/MS Project

Definir si el MVP exporta solo actividades/dependencias o tambien calendarios, recursos y costos.

Gobierno del output IA

Definir estados del cronograma: draft_ai_generated, human_reviewed, approved, exported.

18. Recomendacion final de arquitectura

PlannerAI debe nacer como una plataforma orientada a trazabilidad y control profesional, no solo como un generador automatico de Gantt. Para Aurenza Group, su valor diferencial esta en acelerar la comprension del alcance contractual, transformar documentos tecnicos en planificacion accionable y entregar una salida interoperable con herramientas usadas por PMO, construccion, mineria y EPC/EPCM.

La recomendacion es iniciar con un MVP acotado pero robusto: carga documental, extraccion IA trazable, Gantt editable, ruta critica, tres escenarios y exportacion XML. Una vez validado el flujo, se puede evolucionar hacia recursos, costos, calendarios de turno, contratos multiusuario, bibliotecas de productividad y analitica avanzada.
